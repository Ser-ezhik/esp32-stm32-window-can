# Система окон ESP32 + RP2040 по RS-485

Проект содержит две прошивки:

- `ArduinoIDE/RP2040_RS485_Window_Node_4A` - узел RP2040 на 4 актуатора и 4 датчика тока INA219, связь с ESP32 по RS-485.
- `ArduinoIDE/ESP32_CC1101_RS485_Master` - ESP32-S3 с веб-интерфейсом, CC1101, одной локальной RP2040 по UART и до 8 удаленных RP2040 по RS-485.

## Текущее состояние

RP2040 RS-485 узел реализован полностью:

- 4 актуатора;
- 4 INA219;
- 2 группы питания актуаторов: верх / низ;
- реле направления `выдвижение` / `втягивание`;
- CAP1188;
- 5 входов герконов;
- физические резервные кнопки `Открыто`, `Закрыто`, `Проветривание`;
- сохранение адреса и параметров во Flash;
- смена адреса без перепрошивки по UID через команду RS-485.

ESP32-мастер реализован в `ESP32_CC1101_RS485_Master`:

- CC1101 остается включенным;
- обучение радиопультов остается;
- сохраненные кнопки пультов можно назначать на локальный выход ESP32, локальную RP2040 по UART или любую RP2040 на RS-485;
- одна локальная RP2040 подключается напрямую по UART;
- до 8 дополнительных RP2040 подключаются по RS-485;
- настройки RS-485 узлов находятся на странице `/config` отдельными вкладками;
- адрес узла можно сменить из веб-интерфейса без перепрошивки RP2040.

## RS-485 модуль

Рекомендуется модуль на 3,3 В:

- MAX3485;
- MAX3485E;
- SP3485.

Не использовать 5-вольтовый MAX485 напрямую с ESP32/RP2040.

## Подключение RP2040 к RS-485 модулю

| RP2040 | RS-485 TTL модуль |
| --- | --- |
| GP0 TX | DI |
| GP1 RX | RO |
| GP2 | DE и RE вместе |
| 3V3 | VCC |
| GND | GND |

## Подключение ESP32-S3 к RS-485 модулю

| ESP32-S3 | RS-485 TTL модуль |
| --- | --- |
| GPIO39 TX | DI |
| GPIO38 RX | RO |
| GPIO40 | DE и RE вместе |
| 3V3 | VCC |
| GND | GND |

Локальная RP2040 без RS-485 подключается к ESP32 по UART:

| ESP32-S3 | Локальная RP2040 |
| --- | --- |
| GPIO3 TX | GP1 RX |
| GPIO2 RX | GP0 TX |
| GND | GND |

Линия шины:

```text
A всех модулей вместе
B всех модулей вместе
GND общий
```

Терминатор 120 Ом ставится только на двух физических концах шины.

## INA219

На одной RP2040 RS-485 версии используется 4 INA219 на одной I2C-шине:

```text
0x40
0x41
0x44
0x45
```

## Протокол RS-485

Скорость:

```text
38400 8N1
```

Формат команды:

```text
@<адрес> <команда>
```

Адрес `0` - broadcast.

Команды:

```text
@1 CMD OPEN
@1 CMD CLOSED
@1 CMD VENT
@1 CMD STOP
@1 STATUS
@1 CFG A1MAX=2500 A2MAX=2500 A3MAX=2500 A4MAX=2500 ZEROMA=80 MAXMOVEMS=45000
```

Поиск узлов:

```text
@0 DISCOVER
```

Ответ:

```text
@0 DISCOVER UID=<uid> ADDR=<addr>
```

Смена адреса без перепрошивки:

```text
@0 SETADDR UID=<uid> ADDR=<новый_адрес>
```

RP2040 сохраняет новый адрес во Flash.

### Как менять адрес без перепрошивки

1. Подключите RP2040 к RS-485 шине.
2. Откройте на ESP32 страницу `/config`.
3. Нажмите `Найти узлы`.
4. В строке `DISCOVER` появятся UID и текущий адрес найденного RP2040.
5. В нужной вкладке `RP2040 #...` вставьте UID или дождитесь автозаполнения.
6. Укажите новый адрес.
7. Нажмите `Записать адрес в RP2040`.

Команда уходит broadcast-пакетом:

```text
@0 SETADDR UID=<uid> ADDR=<новый_адрес>
```

Меняет адрес только тот RP2040, чей UID совпал. Новый адрес сохраняется во Flash узла.

## Статус узла

RP2040 отвечает JSON-строкой:

```json
{"addr":1,"uid":"...","state":"idle","target":"none","position":"none","fault":"none","faultActuator":0}
```

## Прошивка RP2040

Сборка через Arduino CLI:

```powershell
arduino-cli compile --fqbn arduino:mbed_rp2040:pico --export-binaries ArduinoIDE/RP2040_RS485_Window_Node_4A
```

Для ручной прошивки используйте `.uf2` из папки `build`.

## Прошивка ESP32-S3

Сборка:

```powershell
arduino-cli compile --fqbn esp32:esp32:esp32s3 --export-binaries ArduinoIDE/ESP32_CC1101_RS485_Master
```

Готовый файл `merged.bin` находится в:

```text
ArduinoIDE/ESP32_CC1101_RS485_Master/build/esp32.esp32.esp32s3/
```

## Веб-интерфейс ESP32

Главная страница `/`:

- `Открыто`;
- `Закрыто`;
- `Проветривание`;
- `Стоп`;
- ссылка `config`.

Страница `/config`:

- настройки локальной RP2040 по UART;
- старые настройки CC1101 и обучение пультов;
- назначение каждой кнопки пульта на действие: локальный выход ESP32, локальная RP2040 UART или конкретный узел RS-485;
- блок `RS-485 RP2040`;
- вкладки для `RP2040 1` ... `RP2040 8`;
- адрес RS-485;
- UID;
- максимальный ток 4 актуаторов;
- порог отсутствия тока;
- максимальное время движения;
- команды `Открыто`, `Закрыто`, `Проветривание`, `Стоп`, `Статус`;
- смена адреса RP2040 по UID.

## Назначение кнопок пульта

После обучения кнопки пульта на странице `/config` у каждой кнопки есть поля:

- `Действие` - куда отправлять команду;
- `Команда` - `Открыто`, `Закрыто`, `Проветривание` или `Стоп`;
- `RS-485` - номер и адрес RP2040, если выбрано действие `RP2040 RS-485`;
- `Лок. выход` - старое поведение, импульс на GPIO ESP32, если выбрано действие `Локальный выход ESP32`.

Так можно сделать, например:

- кнопка 1 пульта управляет локальной RP2040 по UART;
- кнопка 2 управляет RP2040 с адресом `3` на RS-485;
- кнопка 3 остается обычным импульсным выходом ESP32.

Назначения сохраняются во Flash ESP32 вместе с обученными кодами пультов.

## Ограничения текущей версии

- RS-485 протокол текстовый, master-slave. ESP32 опрашивает включенные узлы по очереди.
- Одновременно передавать должен только один узел. Для `DISCOVER` RP2040 добавляет небольшую задержку по адресу, чтобы снизить риск коллизий.
- Для большого количества узлов лучше назначить адреса по одному, затем работать уже адресно.
- Все GPIO ESP32 и RP2040 рассчитаны на 3,3 В.
