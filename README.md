# Система окон ESP32 + RP2040 по RS-485

Проект содержит две прошивки:

- `ArduinoIDE/RP2040_RS485_Window_Node_4A` - узел RP2040 на 4 актуатора и 4 датчика тока INA219, связь с ESP32 по RS-485.
- `ArduinoIDE/ESP32_CC1101_RS485_Master` - ESP32-S3 с веб-интерфейсом, CC1101, двумя локальными RP2040 по UART и до 8 удаленных RP2040 по RS-485.

## Текущее состояние

RP2040 RS-485 узел реализован полностью:

- 4 актуатора;
- 4 INA219;
- 2 группы питания актуаторов: верх / низ;
- реле направления `выдвижение` / `втягивание`;
- CAP1188;
- 5 входов герконов;
- физические резервные кнопки `Открыто`, `Закрыто`, `Проветривание`;
- сохранение адреса и параметров во Flash;
- смена адреса без перепрошивки по UID через команду RS-485.

ESP32-мастер реализован в `ESP32_CC1101_RS485_Master`:

- CC1101 остается включенным;
- обучение радиопультов остается;
- если CC1101 не подключен, Wi-Fi и веб-интерфейс все равно запускаются, но радиопульты временно отключены;
- сохраненные кнопки пультов можно назначать на локальный выход ESP32, локальную RP2040 по UART, одну RP2040 на RS-485 или группу из нескольких RP2040;
- две локальные RP2040 могут подключаться напрямую по UART;
- до 8 дополнительных RP2040 подключаются по RS-485;
- настройки RS-485 узлов находятся на странице `/config` отдельными вкладками;
- адрес узла можно сменить из веб-интерфейса без перепрошивки RP2040.

## RS-485 модуль

Рекомендуется модуль на 3,3 В:

- MAX3485;
- MAX3485E;
- SP3485.

Не использовать 5-вольтовый MAX485 напрямую с ESP32/RP2040.

## Подключение RP2040 к RS-485 модулю

| RP2040 | RS-485 TTL модуль |
| --- | --- |
| GP0 TX | DI |
| GP1 RX | RO |
| GP2 | DE и RE вместе |
| 3V3 | VCC |
| GND | GND |

## Подключение ESP32-S3 к RS-485 модулю

| ESP32-S3 | RS-485 TTL модуль |
| --- | --- |
| GPIO39 TX | DI |
| GPIO38 RX | RO |
| GPIO40 | DE и RE вместе |
| 3V3 | VCC |
| GND | GND |

Локальные RP2040 без RS-485 подключаются к ESP32 по UART:

| ESP32-S3 | Локальная RP2040 #1 |
| --- | --- |
| GPIO3 TX | GP1 RX |
| GPIO2 RX | GP0 TX |
| GND | GND |

| ESP32-S3 | Локальная RP2040 #2 |
| --- | --- |
| GPIO42 TX | GP1 RX |
| GPIO41 RX | GP0 TX |
| GND | GND |

Для второй локальной RP2040 используется `HardwareSerial(0)`. Это рассчитано на ESP32-S3 с монитором порта через USB CDC. Если конкретная плата не выводит GPIO41/GPIO42 или использует их иначе, пины можно заменить в начале прошивки ESP32.

Линия шины:

```text
A всех модулей вместе
B всех модулей вместе
GND общий
```

Терминатор 120 Ом ставится только на двух физических концах шины.

## INA219

На одной RP2040 RS-485 версии используется 4 INA219 на одной I2C-шине:

```text
0x40
0x41
0x44
0x45
```

## Протокол RS-485

Скорость:

```text
38400 8N1
```

Формат команды:

```text
@<адрес> <команда>
```

Адрес `0` - broadcast.

Команды:

```text
@1 CMD OPEN
@1 CMD CLOSED
@1 CMD VENT
@1 CMD STOP
@1 STATUS
@1 CFG A1MAX=2500 A2MAX=2500 A3MAX=2500 A4MAX=2500 ZEROMA=80 MAXMOVEMS=45000
```

Поиск узлов:

```text
@0 DISCOVER
```

Ответ:

```text
@0 DISCOVER UID=<uid> ADDR=<addr>
```

Смена адреса без перепрошивки:

```text
@0 SETADDR UID=<uid> ADDR=<новый_адрес>
```

RP2040 сохраняет новый адрес во Flash.

### Как менять адрес без перепрошивки

1. Подключите RP2040 к RS-485 шине.
2. Откройте на ESP32 страницу `/config`.
3. Нажмите `Найти узлы`.
4. В строке `DISCOVER` появятся UID и текущий адрес найденного RP2040.
5. В нужной вкладке `RP2040 #...` вставьте UID или дождитесь автозаполнения.
6. Укажите новый адрес.
7. Нажмите `Записать адрес в RP2040`.

Команда уходит broadcast-пакетом:

```text
@0 SETADDR UID=<uid> ADDR=<новый_адрес>
```

Меняет адрес только тот RP2040, чей UID совпал. Новый адрес сохраняется во Flash узла.

## Статус узла

RP2040 отвечает JSON-строкой:

```json
{"addr":1,"uid":"...","state":"idle","target":"none","position":"none","fault":"none","faultActuator":0}
```

## Прошивка RP2040

Сборка через Arduino CLI:

```powershell
arduino-cli compile --fqbn arduino:mbed_rp2040:pico --export-binaries ArduinoIDE/RP2040_RS485_Window_Node_4A
```

Для ручной прошивки используйте `.uf2` из папки `build`.

## Прошивка ESP32-S3

Сборка:

```powershell
arduino-cli compile --fqbn esp32:esp32:esp32s3 --export-binaries ArduinoIDE/ESP32_CC1101_RS485_Master
```

Готовый файл `merged.bin` находится в:

```text
ArduinoIDE/ESP32_CC1101_RS485_Master/build/esp32.esp32.esp32s3/
```

## Веб-интерфейс ESP32

Главная страница `/`:

- выбор управляемой RP2040 по имени, например `окно в спальне`;
- `Открыто`;
- `Закрыто`;
- `Проветривание`;
- `Стоп`;
- ссылка `config`.

Страница `/config`:

- имя локальной RP2040 по UART;
- раздельные настройки локальных RP2040 `UART1` и `UART2`;
- старые настройки CC1101 и обучение пультов;
- назначение каждой кнопки пульта на действие: локальный выход ESP32, локальная RP2040 UART или конкретный узел RS-485;
- групповое назначение одной кнопки пульта на несколько RP2040 одновременно;
- блок `RS-485 RP2040`;
- вкладки для `RP2040 1` ... `RP2040 8`;
- адрес RS-485;
- название RS-485 RP2040, которое показывается на главной странице;
- UID;
- максимальный ток 4 актуаторов для каждой RP2040;
- порог отсутствия тока для каждой RP2040;
- максимальное время движения для каждой RP2040;
- настройки защиты CAP1188: включение, маска каналов и задержка подтверждения;
- индикация герконов, CAP1188 и аварии каждого RS-485 узла;
- команды `Открыто`, `Закрыто`, `Проветривание`, `Стоп`, `Статус`;
- смена адреса RP2040 по UID.

## Назначение кнопок пульта

После обучения кнопки пульта на странице `/config` у каждой кнопки есть поля:

- `Действие` - куда отправлять команду;
- `Команда` - `Открыто`, `Закрыто`, `Проветривание` или `Стоп`;
- `RS-485` - номер и адрес RP2040, если выбрано действие `RP2040 RS-485`;
- `Цели группы` - локальная RP2040 и/или несколько RS-485 RP2040, если выбрано действие `Группа RP2040`;
- `Лок. выход` - старое поведение, импульс на GPIO ESP32, если выбрано действие `Локальный выход ESP32`.

Так можно сделать, например:

- кнопка 1 пульта управляет локальной RP2040 по UART;
- кнопка 2 управляет RP2040 с адресом `3` на RS-485;
- кнопка 3 остается обычным импульсным выходом ESP32.

Назначения сохраняются во Flash ESP32 вместе с обученными кодами пультов.

## Универсальная прошивка RP2040

Одна и та же прошивка `RP2040_RS485_Window_Node_4A` понимает оба варианта подключения:

```text
обычный UART: CMD OPEN
RS-485:       @1 CMD OPEN
```

Если команда пришла без `@адрес`, RP2040 считает ее прямой UART-командой и отвечает чистым JSON. Если команда пришла с `@адрес`, RP2040 работает как адресный RS-485 узел и отвечает строкой `@адрес {json}`.

Это позволяет использовать одну прошивку RP2040 и рядом с ESP32 по UART, и удаленно по RS-485. Для сдвоенной двери рядом с ESP32 можно поставить две RP2040 на `UART1` и `UART2`, а остальные двери подключать по общей RS-485 шине.

У локальных RP2040 настройки раздельные: каждая плата имеет свои 4 актуатора, свои пределы тока `A1MAX..A4MAX`, свой порог отсутствия тока, время движения и настройки CAP1188.

Физические кнопки можно подключать параллельно к входам нескольких RP2040 только если это простой общий сигнал `0/3.3 В` с общей землей. TX-выходы UART параллелить нельзя: несколько RP2040 не должны одновременно передавать в один RX ESP32. Для нескольких контроллеров на одной линии используйте RS-485.

## Настройка CAP1188

Для локальной RP2040 и для каждого RS-485 узла на странице `/config` доступны настройки:

- `Защита CAP1188 включена` - если выключить, касание будет отображаться в статусе, но не остановит актуаторы;
- `Маска каналов CAP1188` - какие каналы C1..C8 считаются аварийными;
- `Подтверждение CAP1188, мс` - сколько миллисекунд сработка должна держаться до аварийного стопа.

Маска каналов задается числом от `0` до `255`:

```text
1   = только C1
2   = только C2
3   = C1 + C2
4   = только C3
255 = C1..C8
0   = ни один канал не останавливает движение
```

ESP32 сохраняет эти настройки у себя и отправляет их в RP2040 командой `CFG`. RP2040 сохраняет полученные параметры во Flash.

## Выбор окна на главной странице

На главной странице перед кнопками `Открыто`, `Закрыто`, `Проветривание`, `Стоп` есть список RP2040. В этом списке отображаются понятные имена:

- имя локальной RP2040 задается в блоке `Настройки окон`;
- имена RS-485 RP2040 задаются во вкладках `RP2040 1` ... `RP2040 8`.

Команда с главной страницы отправляется только выбранной RP2040. Выбранное окно сохраняется во Flash ESP32, поэтому после нажатия кнопки или перезагрузки страницы выбор не сбрасывается.

## Ограничения текущей версии

- RS-485 протокол текстовый, master-slave. ESP32 опрашивает включенные узлы по очереди.
- Одновременно передавать должен только один узел. Для `DISCOVER` RP2040 добавляет небольшую задержку по адресу, чтобы снизить риск коллизий.
- Для большого количества узлов лучше назначить адреса по одному, затем работать уже адресно.
- Все GPIO ESP32 и RP2040 рассчитаны на 3,3 В.
