# ESP32 v1.7-safe-ota

Версия добавляет защиту web-обновления ESP32 от случайной или неподходящей прошивки.

## Что изменилось

- Web-обновитель ESP32 больше не принимает сырой `.bin`.
- Для web-обновления используется защищенный контейнер `.wota`.
- В `.wota` есть:
  - магический заголовок `WINESP32OTA1`;
  - назначение `ESP32_CC1101_RS485`;
  - размер payload;
  - CRC32 payload;
  - строка версии.
- Если файл не подходит, ESP32 не начинает запись OTA-раздела.
- После успешной записи ESP32 ставит флаг pending и перезагружается.
- Новая прошивка подтверждает успешный старт только после запуска Wi-Fi/web.
- Добавлена кнопка ручного rollback ESP32 на предыдущий OTA-раздел.

## Файлы

- `ESP32_CC1101_RS485_Master_v1.7.wota` - использовать для дальнейших web-обновлений, когда на ESP32 уже стоит v1.7 или новее.
- `ESP32_CC1101_RS485_Master_v1.7.bin` - использовать для первого перехода со старой web-версии, которая еще не знает формат `.wota`.
- `ESP32_CC1101_RS485_Master_v1.7.merged.bin` - полная прошивка через USB/Serial.

## Важно для первого перехода

Если сейчас на ESP32 стоит старая версия `v1.4-web-update`, ее web-обновитель принимает только обычный `.bin`. Поэтому первый переход на v1.7 делается файлом:

```text
ESP32_CC1101_RS485_Master_v1.7.bin
```

После установки v1.7 все следующие web-обновления ESP32 нужно делать только файлами `.wota`.

## Как создать .wota вручную

```text
python tools/wrap_esp32_ota.py ESP32_CC1101_RS485_Master.ino.bin -o ESP32_CC1101_RS485_Master.wota --version v1.7-safe-ota
```
