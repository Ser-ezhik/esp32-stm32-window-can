# RP2040 v1.6: загрузчик из исходника + основная программа

Это актуальный полный UF2 для первичной прошивки RP2040 через USB-диск.

## Главное

- Загрузчик восстановлен из сохраненного git-коммита `8987d28 Add RP2040 remote OTA bootloader`.
- Основная программа обновлена до `v1.6-ota-bootloader`.
- В основную программу возвращены:
  - команда `BOOTLOADER` для перехода в загрузчик по UART/RS-485;
  - подтверждение успешного запуска после OTA, чтобы загрузчик не делал ошибочный rollback;
  - поддержка OTA-метаданных по адресу `0x101FE000`.

## Файлы

- `RP2040_full_bootloader_app_v1.6.uf2` - полный файл: загрузчик + основная программа. Его использовать для первичной прошивки через BOOT-диск RP2040.
- `RP2040_OTA_Bootloader_v1.6.uf2` - только загрузчик.
- `RP2040_OTA_Bootloader_v1.6.bin` - бинарник загрузчика.
- `RP2040_RS485_Window_Node_4A_appslot_v1.6.uf2` - основная программа, собранная под app-slot `0x10040000`.
- `RP2040_RS485_Window_Node_4A_ota_v1.6.bin` - файл основной программы для удаленного обновления через ESP32.

## Карта flash

```text
0x10000000..0x1003FFFF  bootloader, 256 KB
0x10040000..0x100BFFFF  active app, 512 KB
0x100C0000..0x1013FFFF  staging, 512 KB
0x10140000..0x101BFFFF  backup, 512 KB
0x101FE000              OTA metadata
0x101FF000              настройки приложения
```

Проверка собранного UF2:

- загрузчик: `0x10000000` .. `0x10017C00`;
- приложение: `0x10040000` .. `0x1005C700`;
- пересечений нет.

## Что проверить на стенде

1. Прошить `RP2040_full_bootloader_app_v1.6.uf2`.
2. Убедиться, что RP2040 отвечает ESP32 и показывает версию `v1.6-ota-bootloader`.
3. Отправить команду `BOOTLOADER` и проверить, что контроллер переходит в загрузчик.
4. Выполнить удаленное обновление файлом `RP2040_RS485_Window_Node_4A_ota_v1.6.bin`.
5. Проверить rollback на испорченном образе или прерванной передаче.
